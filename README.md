# fivecorners.bpconditionalactivity

Модуль коробочного Битрикс24, который устанавливает кастомное активити дизайнера
бизнес-процессов **«5 УГЛОВ Запрос с доп. информацией и поддержкой зависимых полей»**
(`CBPRequestInfoDeclineConditional`).

Активити расширяет стандартное `CBPRequestInformationOptionalActivity` и добавляет
**зависимые поля**: когда пользователь выбирает определённое значение в одном поле —
другое поле автоматически появляется и становится обязательным. Скрытие/показ
работает на клиенте (JS), обязательность проверяется и на сервере (PHP).

---

## Назначение

Модуль — тонкая обёртка-установщик. Он не содержит бизнес-логики: вся логика
живёт в самом активити. Задача модуля — установить активити в `/local/activities/`,
подключить нужный JS и зарегистрировать всё это так, чтобы установка и удаление
делались одной кнопкой в `Настройки → Решения → Модули`.

## Два сценария использования

1. **Попап из CRM-сделки.** Бизнес-процесс по сделке доходит до активити —
   пользователю показывается форма с зависимыми полями прямо в карточке сделки.
2. **Вкладка «Автоматизация → Бизнес-процессы»** (`/company/personal/bizproc/NNN/`).
   Та же форма открывается в стандартном интерфейсе задач БП.

Оба пути поддержаны — детали рендеринга см. в [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).

## Требования

- Битрикс24 on-premise (коробочная версия)
- PHP 8.0+
- Модуль `bizproc` (входит в коробку)

## Установка

1. Скопируйте папку модуля в `<bitrix_root>/local/modules/fivecorners.bpconditionalactivity/`.
2. `Настройки → Решения → Модули` (`/bitrix/admin/partner_modules.php`) →
   найдите модуль → **Установить**.
3. После установки активити доступно в дизайнере БП в категории **«Прочее»**
   под названием **«5 УГЛОВ Запрос с доп. информацией и поддержкой зависимых полей»**.

Удаление — там же, кнопкой **Удалить** (двухшаговое, с подтверждением).

## Как работают зависимые поля

В форме настройки активити каждое поле имеет три режима обязательности:

| Режим | Поведение |
|---|---|
| Не обязательно | Всегда видно, не обязательно |
| Всегда обязательно | Всегда видно, всегда обязательно |
| Обязательно при условии | Скрыто; появляется и становится обязательным при выборе нужного значения в триггер-поле |

Для режима **«Обязательно при условии»** указывается поле-триггер (другое поле
этого же активити) и значение, при котором зависимое поле показывается.

Для типа `UF:iblock_element` значение вводится текстом (например, `Низкая ЗП`) —
PHP автоматически резолвит его в числовой ID элемента инфоблока.

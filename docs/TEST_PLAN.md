## Phase 1 — Переоформление в устанавливаемый модуль (v1.0.0)

### BPCO-1-01 — Установка модуля через partner_modules.php

**Given:**
- Папка модуля `fivecorners.bpconditionalactivity` скопирована в `/local/modules/`, модуль ещё не установлен.

**When:**
- Открыть `Настройки → Решения → Модули` (`/bitrix/admin/partner_modules.php`), найти модуль, нажать «Установить».

**Then:**
- Показывается страница `step.php` с зелёным сообщением «Модуль установлен» и пояснением, что активити доступно в дизайнере БП в категории «Прочее».
- Модуль появляется в списке установленных, версия — `1.0.0`, системное имя `fivecorners.bpconditionalactivity`.
- Ошибок установки в админке нет.

**Результат:** ⏸ PENDING

### BPCO-1-02 — Активити появилось в дизайнере БП

**Given:**
- Модуль установлен.

**When:**
- Открыть дизайнер бизнес-процессов, раскрыть категорию «Прочее».

**Then:**
- В списке присутствует активити «5 УГЛОВ Запрос с доп. информацией и поддержкой зависимых полей».
- У активити отображается своя иконка (не дефолтная иконка БП).

**Результат:** ⏸ PENDING

### BPCO-1-03 — Настройка зависимых полей в форме активити

**Given:**
- Активити добавлено в шаблон бизнес-процесса.

**When:**
- Открыть форму настройки активити, добавить запрашиваемые поля, у одного из них выбрать режим обязательности «Обязательно при условии», указать поле-триггер и значение.

**Then:**
- Форма настройки открывается без ошибок, выбор режима «Обязательно при условии» показывает поля «Поле (триггер)» и «Значение».
- Шаблон бизнес-процесса сохраняется, настройки зависимости не теряются при повторном открытии формы.

**Результат:** ⏸ PENDING

### BPCO-1-04 — Зависимое поле в форме задачи БП

**Given:**
- Бизнес-процесс с настроенным активити запущен и дошёл до задачи с зависимым полем.

**When:**
- Открыть форму задачи (попап из карточки CRM-сделки и вкладку «Бизнес-процессы»), в поле-триггере выбрать значение, при котором зависимое поле должно появиться.

**Then:**
- Зависимое поле появляется и помечается как обязательное (звёздочка).
- При попытке отправить форму с пустым зависимым полем — отправка блокируется (проверка на клиенте и на сервере), показывается сообщение об ошибке.
- При значении триггера, не активирующем зависимость, поле скрыто и не обязательно.

**Результат:** ⏸ PENDING

### BPCO-1-05 — Подключение JS на страницах портала

**Given:**

**When:**

**Then:**
- Подключён скрипт `/local/js/fivecorners.bpconditionalactivity/bpridc-conditional.js`, отдаётся с HTTP 200.
- В консоли нет ошибок, связанных со скриптом.

**Результат:** ⏸ PENDING

### BPCO-1-06 — Удаление модуля (двухшаговое)

**Given:**

**Then:**

- Активити пропадает из дизайнера БП.
- Папки `/local/activities/requestinfodeclineconditional/` и `/local/js/fivecorners.bpconditionalactivity/` удалены с диска.
- Модуль снят с регистрации (исчез из списка установленных), кнопка «Удалить» в админке отработала без ошибок.
